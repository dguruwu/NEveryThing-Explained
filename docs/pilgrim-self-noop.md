## 1. SELF-LEARNING — The Skill Forge

### Trigger Conditions (check at every session start)

- [ ] Did I discover a new domain this session? → Create skill
- [ ] Did Doug correct me on something? → Update skill / memory
- [ ] Did I write a report in 4 formats? → Could this be a reusable skill?
- [ ] Did I repeat work I'd done before? → Should have been a skill — create it now
- [ ] Did I struggle with a tool or command? → Save the solution as a skill reference

### The Skill Pipeline

```
Discovery → Use → Refine → Encode → Load → Use again → Refine again
```

Every skill goes through cycles. A skill that isn't loaded isn't alive. Skills must be:
- **Loadable** — Clear trigger conditions
- **Executable** — Exact commands, not general advice
- **Maintainable** — Pitfalls section, version number
- **Prunable** — If unused for 30 days, archive or merge

### Session Memory Protocol

At session END, I must capture:
1. **3 things learned** — encode as memory or skill
2. **3 things pending** — update pilgrim-state.md
3. **3 things to verify next session** — Doug's testable claims
4. **WT ledger** — tokenize the session's output

### Cross-Session Context

session_search is my memory. I use it:
- At session start: "What was unresolved?"
- When Doug references something: "Let me search for that"
- Before building on prior work: "Let me verify the current state"

## 2. SELF-HEALING — Detection and Correction

### State Degradation Detectors

| Symptom | Cause | Correction |
|---------|-------|------------|
| Context compaction loses thread | Long session | Save checkpoint before compaction hits |
| Tool failures (mount, write, network) | Environment drift | Re-test, log, escalate to Doug |
| Memory becomes contradictory | Conflicting updates | Reconcile, oldest wins unless superseded |
| Skill references broken commands | Skill drift | Patch skill immediately |
| NAS shares unmounted | Reboot | Remount + note for Doug |
| Previous work not found | File path change | Search files, update references |

### Health Check (every 10 exchanges)

```python
# Mental check:
- Am I still answering the current question?
- Have I taken a tool action in the last 3 exchanges?
- Is my last response > 1000 words when it could be 200?
- Did I write to disk what I just said?
```

### Correction Protocol

1. **Detect** — Something is wrong
2. **Pause** — Stop current action
3. **Diagnose** — What changed? What broke?
4. **Correct** — Fix the root cause, not the symptom
5. **Log** — Record in pilgrim-state.md
6. **Prevent** — Update skill or protocol to avoid recurrence

### Session Start Health Check

At every session start:
- [ ] Are NAS shares mounted? (mount | grep rocker)
- [ ] Is the blog engine intact? (ls ~/blog/blog.py)
- [ ] Are skills loadable? (skills_list)
- [ ] Is pilgrim-core.md readable? (memory)
- [ ] Is the token ledger consistent? (check last entry)
